# MB85RC v1.0.0

First stable release of the MB85RC256V FRAM library for ESP32-S2 and ESP32-S3 targets using Arduino and PlatformIO.

## Highlights

- Stable public API with deterministic, byte-oriented FRAM access.
- Injected I2C transport layer with no direct `Wire` dependency in the library core.
- Health tracking with `READY`, `DEGRADED`, and `OFFLINE` driver states.
- Device ID verification, rollover-aware read/write/fill behavior, current-address reads, and `verify()` diagnostics.
- One bundled bringup CLI example covering memory inspection, diagnostics, validation, benchmarks, and typed storage demos.
- Native Unity test coverage plus refreshed Doxygen and release documentation.

## Included In This Release

### Added
- Production-ready MB85RC256V driver implementation with chunked transfers and explicit status reporting.
- Runtime settings snapshot support and current-address-read helper coverage.
- Raw and parsed Device ID access.
- Example-only `TypedMemory.h` codec helpers for explicit little-endian fixed-width numeric storage.
- Safe CLI commands for inspection and diagnostics: `hexdump`, `text`, `strings`, `crc`, `verify`, `current`, `probe`, `recover`, `stress`, `stress_mix`, `rw_suite`, `randbench`, and `typed_demo`.

### Fixed
- Transport health updates are routed only through tracked transport wrappers.
- Runtime snapshots are cleared correctly on shutdown and failed re-initialization.
- The example transport supports read-only transactions required by current-address reads.
- `stress_mix` no longer invalidates current-address state before exercising the `currentAddr` path.

### Documentation
- README installation and validation flow updated for the stable `v1.0.0` release.
- Changelog normalized into a dated release entry.
- Public header comments cleaned up for Doxygen output.
